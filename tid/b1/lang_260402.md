# K和forth语言

## kona(k3)

-x选项虽然默认打印，但结果和直觉不符合。`k -x '"hi\n"'`打印出`"hi\n"`，带双引号，同时也不转义。但 <code>k -e '`0: "hi\n"'</code> 就打印出hi并带回车，符合直觉

函数在左，参数在右；从右向左依次执行，沿用数学的`f(g(h(x)))`表示法。

string和symbol差异极大

* `@` 判断原子性，string返回0（不是），symbol返回1
* `0:` 只支持symbol，不支持string
* <code>\`0:</code> 只支持string，要用`$`转换，比如`$1`实现str(1)的效果。反过来写成`0$"1"`则是int("1")。猜测打印是向char设备流式发送字符，symbol作为内部表示不能被拆开，所以要先转成string才能打印
* symbol不能数字开头，会报错parser error

### 读csv

差异过于巨大，颠覆！读一个3列的文件，文档说支持各种类型，实测只能是C，转换为数字时只支持同一类型：

<code>k -x 'd: ("CCC"; ",") 0: `a.csv;v: {{0$x} each x} each d'</code>

* `v 0` 和 `#v` 是列维度，所以`v 0`是第一列，`#v`是3
* `+/ v` 和 `+/ each v`都是取列的相同位置元素求和；`+/' v`是对每列各自求和
